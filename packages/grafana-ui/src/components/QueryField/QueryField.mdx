# QueryField (deprecated)

> **Deprecated.** `QueryField` and the Slate-based editing primitives in `@grafana/ui`
> are deprecated. New code should use [`QueryInput`](?path=/docs/inputs-queryinput--docs)
> instead. These exports may be removed in a future release.

## Background

`QueryField` is built on [Slate](https://docs.slatejs.org/), which is no longer
maintained in `@grafana/ui`. Editing components across Grafana have consolidated
onto [CodeMirror](https://codemirror.net/) so there is a single, unified code
editor to maintain, theme, and extend. `QueryInput` provides the supported query
field API without exposing the editor implementation.

Existing usages keep working — this deprecation is a signal not to build anything
new on Slate, and to migrate existing editors when convenient.

## What's deprecated

- `QueryField`, `QueryFieldProps`
- Typeahead types: `TypeaheadInput`, `TypeaheadOutput`, `SuggestionsState`
- Slate utilities: `SCHEMA`, `makeFragment`, `makeValue`
- Slate plugins: `BracesPlugin`, `ClearPlugin`, `ClipboardPlugin`, `IndentationPlugin`,
  `NewlinePlugin`, `RunnerPlugin`, `SelectionShortcutsPlugin`, `SlatePrism`,
  `SuggestionsPlugin`

## Migrating to QueryInput

`QueryInput` is exported from the main `@grafana/ui` package:

```tsx
import { QueryInput } from '@grafana/ui';
```

### What's different, React-wise

| Concern          | `QueryField` (Slate)                                              | `QueryInput`                                                     |
| ---------------- | ----------------------------------------------------------------- | ---------------------------------------------------------------- |
| Component model  | Class component; mutable Slate `Value` with local two-way binding | Controlled function component; plain `value` + `onChange(value)` |
| Theming          | `Themeable2` — a `theme` prop must be threaded through            | Handled internally                                               |
| Run shortcuts    | `RunnerPlugin` + `onRunQuery`                                     | `onRunQuery` handles Shift+Enter and Ctrl+Enter                  |
| Typeahead portal | required `portalOrigin` prop                                      | not needed                                                       |
| Accessibility    | `contenteditable` div; `aria-labelledby` set manually             | use `aria-label` or `aria-labelledby` on the editor input        |

### Before / after

```tsx
// Before — Slate QueryField
import { QueryField } from '@grafana/ui';

<QueryField query={value} portalOrigin="my-datasource" onChange={setValue} onRunQuery={runQuery} />;
```

```tsx
// After — QueryInput
import { QueryInput } from '@grafana/ui';

<QueryInput value={value} onChange={setValue} onRunQuery={runQuery} aria-label="Query" />;
```
