Org Dashboard Templates — end-to-end architecture

A visual tour of how OrgDashboardTemplate resources are stored, hydrated into scenes, and saved back through the three user flows — plus the OSS/Enterprise seam that keeps the feature out of the OSS bundle. 1Conceptual model — two layers

An OrgDashboardTemplate is a k8s resource whose spec is an outer envelope that wraps a full DashboardV2Spec. All editing flows manipulate the inner layer; only the creation flow sets the outer fields. This split is why save/restore carefully preserves outer fields when writing back.

Why the split matters: diffing two template revisions or restoring an old one only operates on spec.dashboard — otherwise outer-field edits would pollute every diff and a restore could clobber the admin's latest title/tags.

2The three user flows

3OSS ↔ Enterprise seam

The feature is enterprise-only. OSS code cannot import public/app/extensions/, so OSS depends on a small interface (OrgTemplateExtensionHooks). The enterprise bundle registers its implementation at startup; when it isn't loaded, the hooks are a no-op.

6Versions & restore — the embedded-layer trick

Template history comes from the same resource (labeled grafana.app/get-history=true), but each historical item is a full envelope snapshot. To keep diffs meaningful, the transformer stores only spec.dashboard as the revision payload, and restore writes only spec.dashboard back — preserving the admin's latest outer fields.

7Meta flags reference

Two flags on DashboardScene.state.meta gate all template-specific behavior. isOrgTemplate=true means "I'm editing a template"; every other case (including use-template, which is functionally a new dashboard) is isOrgTemplate=false.

FlagTypeMeaningSet when
meta.isOrgTemplate boolean Scene is editing a template resource; save goes to OrgDashboardTemplate PUT. Edit flow only (editTemplate=true).
meta.orgTemplateUid string UID of the template being edited. Required with isOrgTemplate. Edit flow only.
meta.k8s.resourceVersion string Optimistic-concurrency token for the next PUT. Updated after each successful save. Edit flow; refreshed on save and on restore.
meta.isDirty boolean Standard scene-dirty flag. CREATE n/a · USE initial true (primes "Save as new") · EDIT initial false (only real edits mark dirty).

TL;DR — an OrgDashboardTemplate is a two-layer k8s resource (outer envelope + embedded DashboardV2Spec). Three flows act on it: CREATE freezes a dashboard into the inner layer; USE hydrates a fresh scene as a new dashboard; EDIT rewrites only the inner layer via PUT. A small OSS→EE seam (OrgTemplateExtensionHooks) lets OSS branch on two meta flags without importing enterprise code. Only ~10 UI surfaces change; everything else works unchanged because the scene still holds a DashboardV2Spec.
